# eeg-bci-project
# Построение пространства признаков для распознавания управляющих команд по ЭЭГ

Репозиторий содержит программную реализацию экспериментов, выполненных в рамках выпускной квалификационной работы по теме построения пространства признаков для распознавания управляющих команд на основе временных, частотных, частотно-временных и компонентных методов анализа сигналов биоэлектрической активности мозга.

Цель работы — сравнить информативность нескольких семейств признаков ЭЭГ-сигнала и проверить, повышает ли объединение этих признаков качество классификации управляющих команд.

## Состав репозитория

В репозитории размещены пять Jupyter Notebook-файлов:

| Файл                    | Назначение                                                      |
| ----------------------- | --------------------------------------------------------------- |
| `eeg_fft_only.ipynb`    | Извлечение и оценка частотных признаков на основе FFT           |
| `eeg_erp_only.ipynb`    | Извлечение и оценка временных признаков на основе ERP           |
| `eeg_ica_only_v2.ipynb` | Извлечение и оценка компонентных признаков на основе ICA        |
| `eeg_ersp_only.ipynb`   | Извлечение и оценка частотно-временных признаков на основе ERSP |
| `eeg_v25_full.ipynb`    | Объединение всех признаков, классификация и анализ результатов  |

## Используемые методы

В работе рассматриваются четыре семейства признаков:

* **FFT** — частотные признаки, полученные на основе амплитудного спектра;
* **ERP** — временные признаки вызванных потенциалов;
* **ICA** — компонентные признаки, полученные после разложения сигнала на независимые компоненты;
* **ERSP** — частотно-временные признаки, отражающие изменение спектральной мощности относительно базовой линии.

После извлечения признаки объединяются в единое пространство. Далее выполняются нормализация, отбор наиболее информативных признаков, снижение размерности и классификация.

## Классификаторы

Для оценки качества использовались следующие модели:

* Random Forest;
* Support Vector Machine;
* Multilayer Perceptron;
* Logistic Regression;
* Linear Discriminant Analysis;
* Voting Classifier.

Оценка качества выполнялась с использованием групповой кросс-валидации `GroupKFold`, чтобы данные одной группы не попадали одновременно в обучающую и тестовую выборки.

## Основные результаты

Лучшие результаты для отдельных семейств признаков:

| Семейство признаков | Лучшая модель | Accuracy |
| ------------------- | ------------: | -------: |
| FFT                 |           LDA |   56.0 % |
| ERP                 |           LDA |   69.2 % |
| ICA                 |           LDA |   65.6 % |
| ERSP                |           LDA |   60.6 % |

Лучший результат на объединённом пространстве признаков:

| Пространство признаков | Лучшая модель | Accuracy |
| ---------------------- | ------------: | -------: |
| FFT + ERP + ICA + ERSP |           SVM |   79.4 % |

Объединение признаков позволило повысить accuracy на **10.2 процентных пункта** по сравнению с лучшим отдельным семейством признаков.

## Анализ вклада признаков

По результатам анализа наиболее информативных признаков вклад семейств распределился следующим образом:

| Семейство признаков | Доля в top-500 |
| ------------------- | -------------: |
| ERP                 |         51.6 % |
| ERSP                |         33.4 % |
| FFT                 |         10.0 % |
| ICA                 |          5.0 % |

Наибольший вклад в итоговое пространство внесли ERP-признаки, что указывает на высокую информативность временной структуры вызванного ответа.

## Запуск

Ноутбуки рекомендуется запускать в следующем порядке:

1. `eeg_fft_only.ipynb`
2. `eeg_erp_only.ipynb`
3. `eeg_ica_only_v2.ipynb`
4. `eeg_ersp_only.ipynb`
5. `eeg_v25_full.ipynb`

Объединённый ноутбук `eeg_v25_full.ipynb` использует общий пайплайн обработки и предназначен для получения итоговых результатов классификации и визуализаций.

## Структура обработки

Общий pipeline включает следующие этапы:

1. Загрузка ЭЭГ-данных.
2. Предобработка сигнала.
3. Извлечение признаков FFT, ERP, ICA и ERSP.
4. Объединение признаков.
5. Нормализация и отбор признаков.
6. Снижение размерности.
7. Обучение классификаторов.
8. Оценка качества и анализ результатов.

## Примечание

Данные ЭЭГ не включены в репозиторий, если их распространение ограничено условиями использования датасета. Для воспроизведения экспериментов необходимо указать путь к исходным `.set`-файлам в соответствующих ячейках ноутбуков.
